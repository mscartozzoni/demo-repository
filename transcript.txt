Como criar Nameservers para localhost?

[Kodee]
Marcio, criar nameservers personalizados para “localhost” não é possível, pois “localhost” sempre se refere ao seu próprio computador (IP 127.0.0.1) e nunca pode ser usado como nameserver público para domínios na internet.

Se você quer configurar um domínio para apontar para um servidor local (por exemplo, um VPS ou máquina própria), você deve criar registros DNS (como A ou CNAME) apontando para o IP público do seu servidor, não para “localhost”.  
Nameservers precisam ser servidores DNS públicos, com IPs válidos e acessíveis globalmente.

Ótima dúvida, Marcio! Além de separar por subdomínio, você pode organizar seus arquivos de algumas formas diferentes, dependendo da sua preferência e do tamanho do projeto:

### 1. **Todos os sites em uma única pasta**
   - Todos os arquivos de todos os subdomínios ficam juntos em `/var/www/sites`.
   - Não recomendado para vários projetos, pois pode virar bagunça.

### 2. **Pasta principal para cada subdomínio (mais comum)**
   - Como já mostrei: `/var/www/sites/ram`, `/var/www/sites/app`, etc.
   - Cada subdomínio tem seu próprio espaço, facilitando organização e permissões.

### 3. **Pastas separadas por domínio e subdomínio**
   ```
   /var/www/marcioplasticsurgery.com/
       ├── public_html/         (site principal)
       └── ram/
           └── public_html/    (subdomínio ram)
   ```
   - Útil se você tiver vários domínios e subdomínios.

### 4. **Estrutura tipo projetos (para apps com backend/frontend separados)**
   ```
   /var/www/projects/
       ├── ram-frontend/
       └── ram-backend/
   ```
   - Bom para aplicações modernas, onde frontend e backend são separados.

### 5. **Diretórios de usuários (multiusuário)**
   ```
   /home/usuario1/public_html/
   /home/usuario2/public_html/
   ```
   - Usado em hospedagens compartilhadas.

---

**Resumo:**  
O método mais prático e seguro para a maioria dos casos é realmente criar uma pasta para cada subdomínio em `/var/www/sites/nomedoseusubdominio`.

[Kodee]
Claro, Marcio!  
Se você optar pela estrutura de **diretórios de usuários (multiusuário)**, é possível que cada usuário do sistema tenha seu próprio site ou subdomínio, com permissões isoladas. Isso é comum em hospedagens compartilhadas ou ambientes onde cada desenvolvedor tem acesso restrito ao próprio projeto.

### Exemplo de estrutura:
```
/home/usuario1/public_html/
    ├── index.html
    ├── css/
    ├── js/
/home/usuario2/public_html/
    ├── index.html
    ├── assets/
    └── images/
```

### Tipos de usuários:
- **Usuário do sistema:**  
  Cada pasta `/home/usuarioX/` pertence a um usuário Linux (criado com o comando `adduser usuarioX`).  
  Permissões de leitura/escrita são restritas a esse usuário, aumentando a segurança.
- **Usuário do site:**  
  O usuário pode gerenciar apenas os arquivos do seu diretório, sem acesso aos demais.
- **Usuário admin/root:**  
  Tem acesso a todos os diretórios e pode configurar o Nginx, criar usuários, alterar permissões, etc.

### Configuração do Nginx para cada usuário:
```nginx
server {
    listen 80;
    server_name ram.marcioplasticsurgery.com;

    root /home/usuario1/public_html;
    index index.html index.htm index.php;

    location / {
        try_files $uri $uri/ =404;
    }
}
```
- O Nginx aponta o subdomínio para o diretório do usuário correspondente.

### **Vantagens:**
- Segurança: Usuários isolados, sem acesso aos arquivos dos outros.
- Facilidade de gerenciamento individual.
- Ideal para equipes ou hospedagem multi-site.

Se quiser criar usuários ou configurar permissões, posso te passar os comandos e procedimentos!

sudo usermod -aG www-data usuario1

- O usuário pode fazer isso via SFTP, SCP ou diretamente pelo terminal.

Ótima pergunta, Marcio!  
O “perfil” do usuário criado com `adduser` é um usuário padrão do Linux, com as seguintes características:

### Detalhes do perfil do usuário criado:
- **Nome de usuário:** Definido no comando (`usuario1`).
- **Diretório home:** `/home/usuario1`
- **Shell padrão:** geralmente `/bin/bash` (pode ser alterado).
- **Senha:** Definida durante a criação.
- **Grupos:**  
  - Por padrão, o usuário pertence ao próprio grupo (`usuario1`).
  - Pode ser adicionado a outros grupos (ex: `www-data`).
- **Permissões:**  
  - Tem acesso total ao próprio diretório home e subpastas (ex: `/home/usuario1/public_html`).
  - Não tem acesso aos diretórios de outros usuários ou arquivos do sistema protegidos.
- **Arquivo de perfil:**  
  - O usuário possui arquivos de configuração como `.bashrc`, `.profile` em seu diretório home, que definem ambiente, variáveis e preferências do shell.

### Personalização do perfil:
- O usuário pode editar seus próprios arquivos de configuração (ex: `~/.bashrc`) para personalizar o ambiente do terminal.
- O administrador pode definir permissões extras, limitar comandos, ou configurar o ambiente conforme necessário.
